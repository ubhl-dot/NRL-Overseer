NRL Overseer Data Protection & Security Protocol

This Document outlines the strict telemetry logging constraints, storage mechanics, and data protection practices maintained by the NRL Overseer platform. We operate with a policy of minimal data collection, tracking only what is fundamentally critical to calculate player statistics, manage league franchises, and enforce organizational rosters.

1. Data Vector Collection Layout

NRL Overseer programmatically interacts with specific public profile layers to compile system dashboards. The vectors mapped into our backend include:

  • Discord Platform Identifiers: Unique Snowflake IDs, usernames, server channel settings, custom team role configurations, and avatar image metadata handles.
  • Roblox Identity Parameters: Roblox usernames and corresponding numeric account User IDs, isolated strictly via authorized third-party verification webhooks (e.g., Bloxlink API synchronization).
  • Performance Telemetry Metrics: Stat tracking logs parsed directly from official in-game match file uploads (.json/.txt files). This includes player passing, rushing, receiving, kicking, and defensive scoring histories.
  • Transactional Records: Transaction routing structures (logs verifying commercial package activation codes) are recorded for ledger accuracy. Raw credit card numbers or account authentication details are never collected or seen by our systems.

2. Processing Channels & Information Utilization

Data maps are isolated and restricted to the following automated system loops:

  • Generating real-time leaderboard positions, division standings charts, and career telemetry profiles.
  • Processing dynamic team rosters, staff claims, trade confirmations, and automated contract validations.
  • Routing administrative workflows, such as private design support tickets and contextual score parsing channels.
  • Synthesizing game-day recaps and press releases through completely sandboxed narrative queries utilizing client-funded, isolated token credentials.

🔒 Non-Local Secure Infrastructure Matrix

NRL Overseer operates strictly out of dedicated Virtual Private Server (VPS) environments built upon hard-hardened production server baselines.

  • Atomic Replication Sweeps: To guarantee absolute security against local machine failures or environment drops, all active data layers cycle through automated backup engines exactly every 15 minutes, mirroring primary production documents to secure cloud storage partitions.
  • Zero Credential Storage: The codebase contains no components capable of harvesting or recording account passwords, private encryption arrays, or financial payment authorization tokens.

3. Information Distribution & Third-Party Disclosure

Data maps are highly protected assets. Your league registries, player performance matrix rows, and historical document parameters are handled under the following strict boundaries:

  • No External Monetization: Compiled telemetry arrays, usernames, and organization indices are **never** sold, rented, or distributed to outside analytics agencies or third-party marketing firms.
  • System Gateway Sharing: Information is only shared out of absolute programmatic necessity with our verified infrastructure integrations (Discord API handlers, MongoDB Atlas dataspaces, and Supabase cloud replication nodes).

4. Roster Sanitation & Data Retention Right

  • Automated Sanitation Resets: Roster data structures are dropped, altered, or synchronized instantly following server member departures or scheduled league offseason flushes to optimize database lookup speeds.
  • Right to Complete Erasure: Any player or server commissioner maintains the right to request a complete physical purge of their tracking documents. To execute a database wipe request, clear the bot's configurations or open an escalation lane with an initiative system administrator.

5. Updates to this Security Protocol

The platform development studio reserves the privilege to optimize these data security boundaries as required by changes in the Discord API platform rules or Roblox gateway structures. Continued operation of the application implies complete, unconditional acceptance of all active safety protocols.
